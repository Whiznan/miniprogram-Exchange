# 小程序数据表设计

## 总体说明

基于小程序前端功能模块（微信登录、个人主页/编辑、首页 Feed、发布物品、数据中心、搜索），及 BackPlat 现有 Django 后端，抽象出以下数据表。前端目前全量使用 Mock 数据，后端需新增对应模型与 API。

---

## 一、新建数据表

### 1. `User` — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `openid` | CharField(128) | unique | 微信 openid |
| `session_key` | CharField(255) | | 微信 session_key |
| `unionid` | CharField(128) | null, blank | 微信 unionid |
| `nickname` | CharField(64) | | 微信昵称 |
| `avatar_url` | URLField | | 微信头像 URL |
| `gender` | SmallIntegerField | choices=(0,1,2) | 性别：男/女/保密 |
| `location` | CharField(64) | blank | 用户选定地址，如"人才公寓一期" |
| `is_active` | BooleanField | default=True | 是否活跃 |
| `create_time` | DateTimeField | auto_now_add | 注册时间 |
| `update_time` | DateTimeField | auto_now | 更新时间 |


### 2. `Category` — 分类表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `name` | CharField(32) | | 分类名称：二手 / 宠物 / 互助 / 广场 |
| `code` | CharField(32) | unique | 标识码：second-hand / pet / help / square |
| `order` | IntegerField | default=0 | 排序 |
| `create_time` | DateTimeField | auto_now_add | 创建时间 |

### 3. `Post` — 发布表（首页 Feed 卡片）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `user` | ForeignKey | → User, CASCADE | 发布者 |
| `title` | CharField(128) | | 标题 |
| `description` | TextField | | 描述 |
| `category` | ForeignKey | → Category, SET_NULL, null | 分类 |
| `location` | CharField(128) | blank | 位置信息 |
| `status` | CharField(16) | choices=draft/review/published/rejected | 状态 |
| `view_count` | IntegerField | default=0 | 浏览量 |
| `like_count` | IntegerField | default=0 | 点赞量 |
| `share_count` | IntegerField | default=0 | 分享量 |
| `favorite_count` | IntegerField | default=0 | 收藏量 |
| `create_time` | DateTimeField | auto_now_add | 发布时间 |
| `update_time` | DateTimeField | auto_now | 更新时间 |
| `is_delete` | BooleanField | default=False | 软删除 |

### 4. `PostImage` — 发布图片表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `post` | ForeignKey | → Post, CASCADE | 所属发布 |
| `image` | ImageField | upload_to='posts/' | 图片文件 |
| `order` | IntegerField | default=0 | 排序 |
| `create_time` | DateTimeField | auto_now_add | 上传时间 |

## 二、保留的现有数据表

| 表 | verbose_name | 说明 | 是否需要改动 |
|----|-------------|------|------------|
| `Welcome` | 欢迎表 | 启动欢迎页 | 否 |
| `Banners` | 轮播图 | 首页轮播（前端路径 `/home/swipers`） | 否（需 URL 路由映射） |
| `Notice` | 公告表 | 首页公告 | 建议补充 `is_delete` 字段 |

## 三、API 端点对应关系

| 前端请求路径 | 建议后端路由 | HTTP 方法 | 说明 |
|-------------|------------|----------|------|
| `/login/wxLogin` | `app01/wx_login/` | POST | 微信 code 换 JWT token |
| `/api/genPersonalInfo` | `app01/user/profile/` | GET | 获取当前用户个人信息（含相册） |
| `/api/genPersonalInfo` | `app01/user/profile/` | PUT | 更新个人信息 |
| `/home/cards` | `app01/posts/` | GET | 分页获取卡片列表，支持 category 过滤 |
| `/home/cards` | `app01/posts/` | POST | 创建新发布 |
| `/home/swipers` | 复用 `app01/banner/` | GET | 已有接口返回轮播图 |
| `/dataCenter/*` | `app01/data-center/*` | GET | 数据中心（后续实现） |

---

## 四、URL 路由注意事项

前端请求路径不带 `/app01/` 前缀（例如 `/home/cards` 而非 `/app01/home/cards`），而现有后端路由都在 `/app01/` 下。需在 `wx_backPlat/urls.py` 的根 URL 配置中统一注册，或修改前端的 `baseUrl`。
